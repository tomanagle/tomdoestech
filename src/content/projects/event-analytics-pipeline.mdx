## Context & Background

Localz had a microservices architecture where each service would emit it's own set of events to Kafka. There was a service that subscribed to all messages and piped them in to S3 with Kinesis Firehose, we called this the "dirty". The original design used AWS Glue to transform the data into a "clean" bucket that was queries with Athena. The issue was that the data was being displayed on a customer-facing dashboard and the queries would take 60-120 seconds to execute.

## Implementation

The Firehose to the dirty bucket infrastructure was solid, so we kept it. The only change that we made to that flow was to add a PII redactor so no PII made it into the lake.

After reviewing different vendors we landed on Snowflake as the warehouse provider and dbt as the transformation layer. We built a new pipeline that extracted the data from the dirty bucket, transformed it with dbt and loaded it into Snowflake.

## What Went Well
Snowflake was a great choice, the large, complex queries executed in seconds rather than minutes. We could then use the data to surface more complicated insights, such as full customer journeys where the data came from several different services.

## What I'd Do Differently
Glue turned out to be extremely expensive and slow for what we needed. If I was to implement this flow again I would look for tools outside the AWS ecosystem before just settling in familiar territory.
