## Context & Background
M&D was running a Drupal 5 e-commerce website that was slow, difficult to maintain and not mobile-friendly. The goal was to upgrade the website to the latest version of Drupal, which at the time was 7.

## Implementation
The make the website build smooth, we chose a theme and started to build on a local desktop machine that was available to computers inside the office. We customised the theme where necessary with PHP & CSS. We built a customer module to custom module to import products from XML files.

## What Went Well
The website was launched and ran successfully on an AWS EC2 instance. It was mobile-friendly and much faster than the previous version.

At the time there wasn't great options for e-commerce websites that needed custom functionality. Shopify was still fairly immature and WooCommerce didn't provide an easy path for customisation. The team was already failiar with Drupal and so I think sticking to the platform, despite its flaws, was the right choice.
