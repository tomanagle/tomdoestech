## Context & Background

Procurement Leaders ran several large-scale events for procurement professionals. When I joined, the on-the-day content was served to delegates from static websites: the agenda, speakers and sponsors. Each event's pages were built by hand, and there was no room for interactivity.

## Implementation

I built an event hub with Next.js, GraphQL and MongoDB that turned those static pages into something interactive. Delegates could see who else was attending, connect and chat with each other, and give feedback on sessions and speakers during the event. The content (agenda, speakers, sponsors) was ingested automatically from Salesforce, so updates flowed through without anyone rebuilding a page by hand.

## What Went Well

The Salesforce integration meant the content was always current, so the events team no longer had to update the site by hand. And because session feedback was real-time, we got more useful responses: delegates could rate a session while it was still fresh in their minds.

## What I'd Do Differently

I'd build a native app that delegates install before the event so we could use push notifications to remind them to give feedback on sessions, and to let them know when someone has sent them a message. That would have increased engagement and made the networking more effective.
